# Skill — i18n (internacionalização)

> Skill para aplicar i18n corretamente em microsserviços Go
> (`go-i18n/v2`) e frontends Nuxt (`@nuxtjs/i18n`).
> **Idiomas obrigatórios:** `en`, `pt-BR`, `es`.
> Usada por `backend-engineer` e `frontend-engineer`.

---

## Quando carregar

- Vai criar ou modificar uma mensagem de erro da API.
- Vai criar ou modificar copy de UI.
- Vai adicionar uma notificação (e-mail, SMS, push).
- Vai criar uma nova chave de tradução.

---

## Princípio (do `bootstrap.md` §2.11)

> **Toda string externalizada** (mensagem de erro da API, copy de UI,
> e-mail, notificação) **deve** passar pelo sistema de i18n.
> **Nenhuma** string visível ao usuário pode ser hardcoded em código.

### `bundle.go` — carregamento

```go
package i18n

import (
    "embed"
    "encoding/json"

    "github.com/nicksnyder/go-i18n/v2/i18n"
    "golang.org/x/text/language"
)

//go:embed locales/*.json
var localeFS embed.FS

func NewBundle() (*i18n.Bundle, error) {
    bundle := i18n.NewBundle(language.English)
    bundle.RegisterUnmarshalFunc("json", json.Unmarshal)

    for _, loc := range []string{"en", "pt-BR", "es"} {
        if _, err := bundle.LoadMessageFileFS(localeFS, "locales/"+loc+".json"); err != nil {
            return nil, err
        }
    }
    return bundle, nil
}
```

### `translate.go` — helper

```go
package i18n

import (
    "context"

    "github.com/gin-gonic/gin"
    "github.com/nicksnyder/go-i18n/v2/i18n"
)

type ctxKey struct{}

func WithLocalizer(c *gin.Context, defaultLang string) {
    accept := c.GetHeader("Accept-Language")
    lang := parseAcceptLanguage(accept, defaultLang)
    localizer := i18n.NewLocalizer(bundle, lang)
    c.Set("localizer", localizer)
}

func T(c *gin.Context, key string, data ...map[string]any) string {
    loc, ok := c.Get("localizer")
    if !ok {
        return key // fallback: retorna a própria chave
    }
    msg, err := loc.(*i18n.Localizer).Localize(&i18n.LocalizeConfig{
        MessageID:    key,
        TemplateData: mergeData(data...),
    })
    if err != nil {
        return key
    }
    return msg
}

func parseAcceptLanguage(header, defaultLang string) string {
    if header == "" {
        return defaultLang
    }
    // Simplificado — usar http.AcceptLanguage do Go 1.21+ em produção
    tags, _, _ := language.ParseAcceptLanguage(header)
    if len(tags) == 0 {
        return defaultLang
    }
    return tags[0].String()
}

func mergeData(maps ...map[string]any) map[string]any {
    out := map[string]any{}
    for _, m := range maps {
        for k, v := range m {
            out[k] = v
        }
    }
    return out
}
```

### Uso no handler

```go
// internal/handler/auth.go
package handler

import (
    "errors"
    "net/http"

    "github.com/gin-gonic/gin"

    "my-service/internal/api"
    "my-service/internal/i18n"
    "my-service/internal/service"
)

func (h *AuthHandler) Login(c *gin.Context) {
    var req api.LoginRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        c.JSON(http.StatusBadRequest, api.Error{
            Code:    "validation_error",
            Message: i18n.T(c, "validation.invalid_payload", "error", err.Error()),
        })
        return
    }

    res, err := h.svc.Login(c.Request.Context(), req.Email, req.Password)
    if err != nil {
        switch {
        case errors.Is(err, service.ErrInvalidCredentials):
            c.JSON(http.StatusUnauthorized, api.Error{
                Code:    "invalid_credentials",
                Message: i18n.T(c, "auth.invalid_credentials"),
            })
        case errors.Is(err, service.ErrAccountLocked):
            c.JSON(http.StatusForbidden, api.Error{
                Code:    "account_locked",
                Message: i18n.T(c, "auth.account_locked"),
            })
        default:
            c.JSON(http.StatusInternalServerError, api.Error{
                Code:    "internal_error",
                Message: i18n.T(c, "error.internal"),
            })
        }
        return
    }

    c.JSON(http.StatusOK, res)
}
```

## Frontend (Nuxt) — `@nuxtjs/i18n`

### Instalação

```bash
pnpm add @nuxtjs/i18n
```

### `nuxt.config.ts`

```ts
export default defineNuxtConfig({
  modules: ['@nuxt/ui', '@nuxtjs/i18n', '@pinia/nuxt'],
  i18n: {
    strategy: 'no_prefix',
    defaultLocale: 'en',
    locales: [
      { code: 'en',    name: 'English',     file: 'en.json' },
      { code: 'pt-BR', name: 'Português',   file: 'pt-BR.json' },
      { code: 'es',    name: 'Español',     file: 'es.json' },
    ],
    detectBrowserLanguage: {
      useCookie: true,
      cookieKey: 'i18n_redirected',
      redirectOn: 'root',
      fallbackLocale: 'en',
    },
  },
})
```

### Uso em templates

```vue
<script setup lang="ts">
const { t, locale, locales, setLocale } = useI18n()
</script>

<template>
  <div>
    <h1>{{ t('home.welcome') }}</h1>
    <p>{{ t('auth.invalid_credentials') }}</p>

    <!-- Seletor de idioma -->
    <USelect
      :model-value="locale"
      :items="locales.map(l => ({ label: l.name, value: l.code }))"
      @update:model-value="(v: string) => setLocale(v as any)"
    />
  </div>
</template>
```

## Convenção de chaves

Use **namespace** claro:

```
<domínio>.<ação>.<contexto>
```

Exemplos:
- `auth.invalid_credentials` (não `error1` ou `unauthorized`)
- `user.created`
- `user.delete_blocked`
- `validation.invalid_email`
- `error.internal`
- `home.welcome`

> **NUNCA** use chaves planas (`error1`, `unauthorized`). O sensor
> `08-i18n-audit` detecta isso.
